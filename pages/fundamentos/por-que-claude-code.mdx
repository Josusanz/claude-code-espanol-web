# Leccion 1.2: ¿Por qué Claude Code?

## El panorama en 2026

Las herramientas de IA para programar han **explotado**. Hay IDEs inteligentes, CLIs, plataformas completas, generadores de apps... El 85% de los desarrolladores ya usa alguna herramienta de IA en su día a día. Y el 78% de las organizaciones las integra en sus flujos de trabajo.

Con tantas opciones, es normal preguntarse: **¿por qué elegir Claude Code?**

Vamos a verlo.

## Las opciones que existen

Aquí tienes un mapa rápido y honesto del ecosistema actual:

| Herramienta | Tipo | Ideal para | Limitación principal |
|---|---|---|---|
| **Cursor 2.0** | IDE con IA | Devs que prefieren interfaz visual | Atado a su editor, menos flexible |
| **GitHub Copilot** | Autocompletado + agente | Sugerencias rápidas dentro del IDE | Menos autónomo en tareas complejas |
| **Replit Agent** | Plataforma completa | Principiantes que quieren todo integrado | Menos control, dependes de su plataforma |
| **Bolt.new** | Generador de apps | Prototipos rápidos | Apps genéricas, poco personalizable |
| **v0.dev** | Generador de UI | Componentes bonitos rápidamente | Solo interfaz visual, no apps completas |
| **Windsurf** | IDE agéntico | Equipos grandes, multi-repositorio | Ecosistema todavía pequeño |
| **Codex 5.3** | CLI de OpenAI | Velocidad y volumen de código | Menos profundo en razonamiento complejo |
| **Claude Code** | CLI agéntico | Control total, autonomía, flexibilidad | Requiere terminal (¡que enseñamos aquí!) |

<Callout type="info">
  **Todas estas herramientas son buenas.** No se trata de tirar piedras a nadie. Cada una brilla en su contexto. Pero para aprender a construir software real con máximo control, Claude Code tiene ventajas únicas.
</Callout>

## ¿Qué hace especial a Claude Code?

### 1. La terminal = poder sin límites

Claude Code vive en tu terminal. Eso significa que **no dependes de ningún editor específico**. Funciona igual con VS Code, Neovim, JetBrains, Emacs o cualquier otro editor que uses.

Mientras otras herramientas te atan a su entorno, Claude Code se adapta al tuyo.

### 2. Agent Teams: trabajo en paralelo

¿Necesitas hacer varias cosas a la vez? Claude Code puede lanzar **múltiples agentes trabajando en paralelo**. Uno refactoriza, otro escribe tests, otro actualiza documentación. Todo al mismo tiempo.

Es como tener un equipo entero de asistentes.

### 3. Personalización total con Skills, Hooks y Plugins

Claude Code no es una caja negra. Puedes configurarlo a tu medida:

- **Skills**: Enséñale comportamientos específicos para tu proyecto
- **Hooks**: Automatiza acciones que se disparan con eventos
- **Plugins**: Extiende sus capacidades con herramientas externas

### 4. MCP: conecta con cualquier herramienta

El Model Context Protocol (MCP) permite que Claude Code se comunique con **cualquier servicio externo**: bases de datos, APIs, herramientas de diseño, sistemas de deploy...

Es como darle superpoderes de integración.

### 5. 1 millón de tokens de contexto

Claude Code puede mantener en su cabeza hasta **1 millón de tokens de contexto**. Eso significa que puede entender proyectos enormes sin perder el hilo. Mientras otras herramientas se confunden con archivos grandes, Claude Code mantiene la visión completa.

### 6. Es la herramienta de los profesionales

Los desarrolladores más productivos del mundo están usando Claude Code. No es casualidad: la combinación de autonomía, contexto masivo y flexibilidad lo convierte en la herramienta más potente disponible.

## El paradigma del "Vibe Coding"

Hay un concepto que lo cambia todo: el **vibe coding**.

El término lo acuñó Andrej Karpathy (cofundador de OpenAI, ex-director de IA en Tesla) para describir una nueva forma de programar:

> En lugar de escribir cada línea de código, **describes tu intención** y la IA construye por ti.

Esto cambia tu rol fundamentalmente:

| Antes | Ahora |
|-------|-------|
| Escribes código línea a línea | Describes lo que quieres |
| Buscas en StackOverflow | Claude busca la solución |
| Depuras errores manualmente | Claude diagnostica y arregla |
| Eres el albañil | Eres el **arquitecto** |

<Callout type="warning">
  **Importante:** Vibe coding no significa "no entender nada". Los mejores resultados vienen cuando sabes **qué pedir** y **cómo verificar** que lo que se construyó está bien. Este curso te enseña exactamente eso.
</Callout>

Con Claude Code, te conviertes en un **orquestador**: defines la visión, diriges el trabajo y revisas los resultados. Es desarrollo basado en intención, y es el futuro.

## ¿Por qué Claude Code para este curso?

Podríamos haber elegido cualquier herramienta. Elegimos Claude Code porque:

- **Enseña los fundamentos correctos.** La terminal y el pensamiento en comandos son habilidades transferibles a cualquier herramienta futura.
- **No te ata a nada.** Lo que aprendas aquí funciona en cualquier entorno, con cualquier editor.
- **Es el más capaz.** Para tareas complejas y razonamiento profundo, Claude está en otro nivel.
- **Crece contigo.** Desde un "hazme un archivo" hasta orquestar equipos de agentes con MCP, no le queda chico.
