# Lección 3: Servidores MCP populares

Existen **miles** de servidores MCP creados por la comunidad y las grandes empresas tecnológicas. Aquí están los más útiles organizados por categoría.

<Callout type="info">
  La mayoría de estos servidores son **gratuitos y de código abierto**. En 2026, el ecosistema explotó: Microsoft, AWS y Google lanzaron servidores oficiales, y es cada vez más común que las herramientas vengan con soporte MCP de serie.
</Callout>

---

## Servidores oficiales de Anthropic

Estos están mantenidos directamente por el equipo de Anthropic:

| Servidor | Descripción | Uso típico |
|----------|-------------|------------|
| **filesystem** | Acceso a archivos locales | Leer/escribir archivos en tu computadora |
| **git** | Operaciones de Git | Commits, branches, historial |
| **fetch** | Obtener contenido web | Leer páginas web, APIs |
| **memory** | Memoria persistente | Recordar información entre sesiones |
| **sequential-thinking** | Razonamiento paso a paso | Problemas complejos |

Repositorio oficial: [github.com/modelcontextprotocol/servers](https://github.com/modelcontextprotocol/servers)

## Cloud e infraestructura (2026)

Las grandes plataformas cloud ahora tienen servidores MCP oficiales:

### AWS
- **ECS/EKS** - Gestionar contenedores y Kubernetes
- **Lambda/Serverless** - Desplegar funciones
- **CloudWatch** - Monitorización y logs

### Google Cloud
- **Maps** - Geolocalización y rutas
- **BigQuery** - Análisis de datos masivos
- **Compute Engine** - Gestionar VMs
- **Kubernetes Engine** - Orquestar contenedores

### Microsoft Azure
- **40+ servicios** integrados
- Conectado con **Copilot Studio**
- Gestión de recursos, monitorización, deploy

<Callout type="info">
  Esto significa que puedes pedirle a Claude: "Despliega esta función en AWS Lambda" o "Muéstrame los logs de errores en CloudWatch" directamente desde tu terminal.
</Callout>

## Cómo elegir un servidor

<Callout type="warning">
  No instales todos los servidores. Solo los que realmente vayas a usar. Cada servidor consume recursos.
</Callout>

**Preguntas para elegir:**

1. **¿Qué herramienta uso más?** → Empieza por esa
2. **¿Es un servidor oficial o de comunidad?** → Oficiales más estables
3. **¿Tiene buena documentación?** → Revisa el README
4. **¿Está activo el desarrollo?** → Revisa commits recientes
