# Lección 1: ¿Qué es MCP?

## El problema que resuelve MCP

Imagina que quieres que Claude:
- Lea tus notas de Notion
- Consulte tu base de datos
- Cree archivos en tu Google Drive
- Revise tus repos de GitHub

**Sin MCP:** Cada herramienta necesita su propia integración personalizada. Caos total.

**Con MCP:** Un protocolo universal. Conectas una vez, funciona con todo.

<Callout type="info">
  MCP es como el **USB de las IAs**. Antes cada dispositivo tenía su propio cable. Ahora todo usa USB. MCP hace lo mismo para conectar IAs con herramientas.
</Callout>

---

## ¿Qué significa MCP?

**Model Context Protocol** - Un estándar abierto creado por Anthropic que define cómo las IAs se comunican con herramientas externas.

| Antes de MCP | Con MCP |
|--------------|---------|
| Cada IA tiene su propia forma de conectar | Un protocolo universal para todas |
| Los desarrolladores reinventan la rueda | Se reutilizan "servidores MCP" |
| Integraciones frágiles y diferentes | Conexiones estándar y confiables |

---

## La analogía del restaurante

Piensa en MCP como el sistema de un restaurante:

- **El cliente (tú):** Pides lo que quieres
- **El mesero (Claude):** Entiende tu pedido y lo comunica
- **El protocolo MCP:** El sistema de comandas estándar
- **La cocina (servidores MCP):** Notion, GitHub, bases de datos...

El mesero no necesita saber cocinar. Solo necesita hablar el mismo idioma que la cocina.

---

## ¿Quién usa MCP?

MCP fue creado por Anthropic en noviembre 2024 y en poco más de un año se convirtió en **el estándar universal de la industria**:

**Creadores de IA:**
- **Anthropic** - Claude y Claude Code
- **OpenAI** - GPT y Codex
- **Google DeepMind** - Gemini y herramientas de Google

**Grandes plataformas cloud:**
- **Microsoft** - Integrado en Copilot Studio, soporte para 40+ servicios de Azure
- **AWS** - Servidores oficiales para ECS, EKS, Lambda y Serverless
- **Google Cloud** - Maps, BigQuery, Compute Engine, Kubernetes Engine

**Herramientas de desarrollo:**
- Cursor, Zed, Sourcegraph, Windsurf, Replit, Block

<Callout type="warning">
  En diciembre 2025, Anthropic donó MCP a la **Linux Foundation** bajo la nueva **Agentic AI Foundation**, convirtiéndolo en un estándar abierto gobernado por la comunidad — al mismo nivel que Linux o Kubernetes.
</Callout>

## Novedades del protocolo (2026)

MCP ha evolucionado mucho desde su lanzamiento. Estas son las capacidades más recientes:

### MCP Apps (enero 2026)
Los servidores ahora pueden enviar **elementos de UI interactivos** que se renderizan directamente en la conversación. Claude puede mostrarte formularios, tablas y controles sin salir del chat.

### Soporte multimedia
MCP ahora soporta **imágenes, video y audio** además de texto. Los agentes pueden ver, escuchar y procesar contenido multimedia.

### Mensajes chunked
Para outputs largos, el protocolo ahora soporta **streaming por chunks** y **streams multipart** que mezclan distintos tipos de datos.

### Streamable HTTP
Una evolución del transporte SSE que permite comunicación bidireccional más eficiente entre cliente y servidor.

## ¿Qué puede hacer MCP?

Los servidores MCP exponen **tres tipos de capacidades**:

### 1. Herramientas (Tools)
Acciones que Claude puede ejecutar:
- Crear un archivo
- Enviar un mensaje
- Ejecutar una consulta SQL

### 2. Recursos (Resources)
Datos que Claude puede leer:
- Contenido de archivos
- Registros de base de datos
- Páginas de Notion

### 3. Prompts
Plantillas predefinidas para tareas comunes:
- "Analiza este código"
- "Resume este documento"

## Ejemplo real

Sin MCP:
```
Tú: "Claude, ¿puedes revisar mis tareas de Notion?"
Claude: "No tengo acceso a Notion. Copia y pega el contenido aquí."
```

Con MCP + servidor de Notion:
```
Tú: "Claude, ¿puedes revisar mis tareas de Notion?"
Claude: *lee directamente tu Notion* "Tienes 5 tareas pendientes..."
```
